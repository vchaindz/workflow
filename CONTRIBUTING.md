# Contributing to workflow

Thanks for your interest in contributing!

## Getting started

1. Fork and clone the repo
2. Run `cargo build` to verify the build
3. Run `cargo test` to run all tests
4. Run `cargo clippy` to check for lint warnings

## Submitting changes

- Open an issue first for large changes so we can discuss the approach
- Keep PRs focused — one feature or fix per PR
- Add tests for new functionality (see `tests/cli_tests.rs` for examples)
- Run `cargo test && cargo clippy` before submitting

## Reporting bugs

File a GitHub issue with:
- What you expected to happen
- What actually happened
- Steps to reproduce
- Your OS and workflow version (`workflow --version`)

## Code of conduct

Be kind. Be constructive. We're all here to build useful tools.
