## Codebase Patterns
- Features in Cargo.toml use `dep:` syntax for optional dependencies (e.g., `mcp = ["dep:rmcp", "dep:tokio"]`)
- serde_json is already a non-optional dependency — no need to add it for MCP args
- All tests (unit + integration) run via `cargo test`; integration tests use `assert_cmd` + `tempfile::TempDir`
- When adding a new field to `Step` struct, ALL construction sites must be updated (parser.rs, executor.rs, wizard.rs, tests/cli_tests.rs) — use `sed -i` for bulk changes
- `McpStepConfig` and `McpServerRef` types live in `models.rs` and are NOT feature-gated (always available); only execution logic is feature-gated
